class sve.sve.Sve(rIni, rMin, rMax, stop, dMax=inf, dBox=False, debug=False)[source]¶

Bases: object

A python class for the simplex vertex-edge optimizer.

The SVE method seeks to find a stationary point x[0..n-1] of the n-argument function F by dynamically adjusting random probe simplexes centered on the current guess x*[0..n-1].

rIni : float
Initial simplex radius
rMin : float
Minimum simplex radius
rMax : float
Maximum simplex radius
stop : float, default inf
The Maximum distance over sucessive guesses (x).
dBox : bool, default False
If True, indicates that the search domain is an an n-dimensional cube with side 2*`dMax`. If False, the domain is a ball with radius dMax.
dMax : float , default inf
Maximum search domain value. The half-side of a cube or the radius of a ball, according to dBox.
debug : bool , default False.
Print diagnostic messages if True.

The parameters of the SVE object are basically the various distances (rIni, rMin, rMax) that characterize a simplex, as well as the optimization direction (direction), the stop criteria (stop), and information regarding the search domain (dBox and dMax).

Iterative quadratic optimization by the simplex vertex-edge method

Parameters:
  • x0 (list [ float ], (n,)) – Initial guess
  • F (callable) – Function to be optimized, with signature F(x, *fparams). It should compute some function of the point x[0..n-1], with optional arguments (fparams)
  • fparams (list, optional) – A list of optional arguments to pass to F, by default an empy list [].
  • direction ({‘NEG’, ‘POS’, ‘ZERO’}) – Optimization direction. If ‘POS’, the procedure looks for a local maximum of {F}. If ‘NEG’, it looks for a local minimum. If ‘ZERO’, a stationary point of {F} is sought.
  • maxIters (int, optional, by default 300) – Maximum number of iterations
  • OK (callable, optional, by default None.) – Optional function that implements a given stop criteria. Need to have the signature OK(n, x, Fx). If True is returned, the iterative process ends.
  • stop_if_dFx_lt_tol (bool, default False) – If True, stop the iterative procedure when the maximum absolute diference between successive guesses F(x*) is less then tol_dfx.
  • tol_dfx (float, by default 1e-3.) – Maximum absolute difference over successive guesses of the function value F(x*).
Returns:

outdata – The output data has the following entries:

X

A numpy.ndarray of dimension (niter, n+1) with the values of the optimal guesses (x*) at each iteration.

Fx
A numpy.ndarray of dimension (niter,) with the values of the

Fx(x*) at each iteration.

"v"

A numpy.ndarray of dimension (niter,n*(n+1)) with the simplex centers at each iteration.

"nit"

Number of iterations (int) performed by the method.

"nit"

Number of functino evalueations (int).

"maxiter"

Maximum number of iterations considered.
Return type:

dict

Notes

At each iteration, the SVE method chooses a random probe simplex centered on the current guess {x[0..n-1]}. The radius {r} of the simplex is dynamically adjusted, starting with {rIni} but staying within the range [ rMin _ rMax].

The iterations will stop when: A) the current guess {x} satisfies the predicate {OK(n,x,F(n,x))} B) the distance between successive guesses (delta x*) is less than stop C) the quadratic minimization loop has been performed maxIters times. D) the flag stop_if_dFx_lt_tol is True, and delta F(x*) is less then ` tol_dfx`.
